\doxysubsection{Detailed Description}
\mbox{\hyperlink{classmicrosoft_1_1quantum_1_1_i_operand_prototype}{IOperand\+Prototype}} describes an IR pattern and allows matching against LLVMs llvm\+::\+Value type. Each value may or may not be captured during the matching process which means that they are stored in a map under a given name. Capturing is enabled using {\ttfamily capture\+As(name)} which sets the name the value should be stored under.

Interface function which determines if a given Value matches the implemented pattern. It is expected that any implementation of {\ttfamily match} will return a call to either {\ttfamily \mbox{\hyperlink{classmicrosoft_1_1quantum_1_1_i_operand_prototype_af9cf2561fc7d6bb39ec9f9486156d133}{success()}}} or {\ttfamily \mbox{\hyperlink{classmicrosoft_1_1quantum_1_1_i_operand_prototype_a25043289d7efcbe79db466f49dd51cd2}{fail()}}}. These functions will, in turn, ensure that the node is captured in the capture table (and erased upon backtracking) as well as matching children.
